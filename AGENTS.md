# Agent Instructions

For any substantive work in this repository, load these files into context first:

1. `ENGINEERING_PLAYBOOK.md`
2. `REPO_WORK_RULES.md`
3. `DELIVERY_CHECKLIST.md`

Then use existing repo docs as needed:

- `REPO_BASELINE.md` for recommended repo setup and standard tooling expectations
- `README.md` for entrypoint usage and operator-facing workflow
- `CODEX_PROJECT.md` for runtime contracts
- `CODEX_STRUCTURE.md` for code layout
- `DECISIONS.md` for ADRs
- `docs/diagrams/` for current operational diagrams

## Working Expectations

- Treat `ENGINEERING_PLAYBOOK.md` as reusable policy.
- Treat `REPO_WORK_RULES.md` as repository-specific rules.
- Treat `DELIVERY_CHECKLIST.md` as the minimum close-out checklist before declaring work complete.
- Treat `REPO_BASELINE.md` as the standard setup and tooling reference for new or normalized repos.
- Keep docs and diagrams aligned with code changes in the same change set.
